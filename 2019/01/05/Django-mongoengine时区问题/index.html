Django mongoengine时区问题

Jan 5, 2019 | 阅读

背景
之前所有的项目使用的是mongoengine来操作mongodb，直接使用datetime.now()来获取本地时间，存入数据库。Django REST 服务直接读出来返回给前端，时间没有问题。数据库、程序、前端时间是一致的。后来，数据服务采用Go编写，发现使用time.Now()获取的时间存入mongodb会被转成UTC时间，少8个小时，此时Django REST服务器读取出来传给前端的时间也少了8个小时。
