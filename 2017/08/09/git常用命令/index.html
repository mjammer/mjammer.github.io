git常用命令

Aug 9, 2017 | 阅读

远程仓库相关命令

检出仓库 git clone git://github.com/jquery/jquery.git
查看远程仓库 git remote -v
添加远程仓库 git remote add [name] [url]
删除远程仓库 git remote rm [name]
拉取远程仓库 git pull [remoteName] [localBranchName]
推送远程仓库 git push [remoteName] [localBranchName]

分支(branch)操作相关命令

查看本地分支 git branch
查看远程分支 git branch -r
创建本地分支 git branch [name] —-注意新分支创建后不会自动切换为当前分支
切换分支 git checkout [name]
创建新分支并立即切换到新分支 git checkout -b [name]
删除分支 git branch -d [name] —- -d选项只能删除已经参与了合并的分支，对于未有合并的分支是无法删除的。如果想强制删除一个分支，可以使用-D选项
合并分支 git merge [name] —-将名称为[name]的分支与当前分支合并
创建远程分支(本地分支push到远程) git push origin [name]
删除远程分支 git push origin :heads/[name]
版本(tag)操作相关命令

查看版本 git tag
创建版本 git tag [name]
删除版本 `git tag -d [name]
查看远程版本 git tag -r
创建远程版本(本地版本push到远程) git push origin [name]
删除远程版本 git push origin :refs/tags/[name]
子模块(submodule)相关操作命令

添加子模块 1git submodule add [url] [path]
初始化子模块 git submodule init —-只在首次检出仓库时运行一次就行
更新子模块 git submodule update —-每次更新或切换分支后都需要运行一下
删除子模块 git rm –cached [path]
忽略一些文件、文件夹不提交

在仓库根目录下创建名称为“.gitignore”的文件，写入不需要的文件夹名或文件，每个元素占一行即可，如 target bin *.db

回滚相关

修改最近一次提交的注释 git commit –amend 进入编辑模式
将某次错误的改变提交了，常用的有如下两种方式回滚:
git reset --hard HEAD^ 回滚最近一次提交
git reset --hard commitxxx回滚至某次提交
有时候运用了第二个操作之后又想回到最近的某次提交，这个时候用git log查不到commitxxx，这个时候使用git reflog查看即可，这个日志保留的是对版本库的所有操作。

回退某个文件到某次提交
1、使用git log filename查看某个文件的所有提交。
2、git reset commitxxx filename回滚。
3、提交. git commit
4、git checkout filename
经过以上四步，某个文件才算真正回滚成功。

不小心将某次提交推送到远端。
1、git reset –hard
2、git push origin HEAD –force

撤销指定的某次提交
git revert commitxxx

重写历史。误将某个不属于版本库中的文件提交，该文件可能非常大，会使版本库变得异常大。这个时候简单的使用以上操作并不能真正的将文件从版本库中删除，这个时候就要使用一下操作进行删除。
1、git filter-branch –tree-filter ‘rm -rf filename’ –tag-name-filter cat – –all
2、git push origin –tags –force 3、git push origin –all –force

使用pull强制覆盖本地文件。注意该操作会将本地所有文件覆盖掉
1、git fetch –all
2、git reset –hard origin/master

暂存
需要在某个分支下进行开发，但此时这个分支下还有很多文件修改过，此时不能提交。这个时候就可以使用git中的暂存功能了。
git stash 将不能提交的修改暂存或 git stash save 注释
git stash pop 将暂存的修改取出或
git stash apply @xxx
git stash list 查看所有暂存

冲突
1、普通文本文件冲突。按照常规的方法解决冲突，提交即可。
2、二进制文件冲突。二进制冲突，一般采用覆盖方式，要么保留版本库中的文件，要么保留本地文件，采用如下命令可以做到这点：
git checkout –theirs YOUR_BINARY_FILE // git checkout –ours YOUR_BINARY_FILE
git add YOUR_BINARY_FILE
git commit
git push
